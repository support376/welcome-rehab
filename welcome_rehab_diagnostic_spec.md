# 웰컴법률사무소 개인회생 AI 진단기 설계 명세서

> 회생 진단기 + 상담 예약 퍼널의 설계·개발 인계용 문서.
> 본 문서만으로 와이어프레임 / AI 봇 프롬프트 / 백엔드 로직 / 운영 룰을 구성할 수 있도록 작성됨.
> 작성 기준일 기준 적용 데이터: 2026년 보건복지부 기준 중위소득.

---

## 0. 한 줄 요약

광고 → 랜딩 → **탭 5개 (30초 추정)** → **AI 챗봇 (체크박스 + 동적 분기)** → **케이스 자동 분류 (Lite / Pro)** → **상담 예약 (2만원 노쇼방지비) → 30분 무료 전화상담 → 수임**

---

## 1. 타겟 & 전략 컨셉

### 1.1 두 타겟 동시 흡수
- **저가형 타겟**: 단순 직장인·일용직, 채무 단순. 30초 탭만으로 결과 → 즉시 상담 예약. 챗봇 스킵 가능.
- **복잡형 타겟**: 자영업·전문직·법인·재산 보유. 챗봇으로 정밀 진단 → Pro 추천.

### 1.2 핵심 차별점
- **본인이 입력 → 본인이 결과 확인**이 회생 시장 전환의 핵심 (자기 상황 자각 = 가장 강한 상담 동기).
- **AI 챗봇 대화 내용 기반 웰컴 강점 노출** (사용자 키워드 매칭 사례 제시).
- **2만원 노쇼방지비**: 무료 50~60% 노쇼율 → 2만원이면 5~10%로 떨어지는 골든존. "상담료"가 아닌 "약속금" 프레이밍.

### 1.3 진입 경로
**랜딩 페이지 우선** (③안). 메타 광고 / 검색 광고 → 랜딩 진입 → 시뮬레이터 즉시 사용 가능.
- 광고 카피 예: "월 얼마 갚아도 되는지 1분 진단", "개인회생 가능한지 무료 확인"
- 메타 리드폼 사용 안 함. 진단 결과 직후 카톡 PDF 발송 시점에 전화번호 수집.

## 2. 변제금 계산 로직 (백엔드 핵심)

### 2.1 변제금 결정 3원칙
법원 결정 총 변제금 = **다음 셋 중 가장 큰 금액**

```
1. 가용소득 기준
   월 변제금 = 월 소득 - (기본생계비 + 추가생계비 + 기타생계비)
   총 변제금 = 월 변제금 × 36개월 (또는 60개월)

2. 청산가치 보장 원칙
   총 변제금 ≥ 청산가치
   청산가치 = (모든 재산 - 면제재산) - 담보채권 (담보재산의 경우)

3. 최저변제액 원칙
```

| 2인 | 4,199,292원 | **약 2,519,575원** |

### 2.3 추가생계비 인정 항목

#### 주거비 (4권역 차등)
| 권역 | 범위 |
|---|---|
| 1권역 | 서울 전 지역 |
| 2권역 | 서울 제외 수도권 과밀억제권역, 세종, 용인, 화성 |
| 3권역 | 광역시 (군 단위 제외) — 부산·대구·광주·대전·울산 |
| 4권역 | 그 외 지역 |

- 인정 대상: 월세 / 주담대 원리금 / 전세자금대출 이자 / 관리비 (서울회생법원 확대)
- 권역별 한도: 서울회생법원 매년 발표 기준 적용

#### 교육비
- 미성년 자녀 일반 교육비: 자녀 1명당 약 20만원
- 특수교육 (장애): 50만원 초과도 인정 가능 (증빙 시)

#### 의료비
- 본인·부양가족의 지속적 의료비
- 기준 중위소득 60%에 포함된 의료비 초과분만 추가 인정
- 1인 가구 기준 약 6만원 초과분부터 인정

#### 부양가족 인정
- **배우자**: 과거 1년 소득 100만원 이하 (근로소득만일 경우 총급여 500만원 이하), 육아·간병으로 일 못 함
- **자녀 (만 21세 미만 성년)**: 무소득 또는 소득 100만원 이하 (근로소득만일 경우 총급여 500만원 이하)
- **65세 이상 부모**: 부양 정황 입증 시
- 미성년 자녀: 자동 인정

#### 기타생계비
- 월 소득이 기준 중위소득 150% 초과 채무자에게만 인정
- 기본+추가+기타 합계 ≤ 중위소득 100%
- 변제율 40% 미만 또는 최근 6개월 채무가 총채무 50% 초과 시 불인정

### 2.4 청산가치 산정

#### 합산 재산
- 부동산 (시가 - 근저당)
- 자동차 (중고시세)
- 예금·적금
- 보험 해약환급금
- 퇴직금 (예상액)
- 임차보증금
- 주식·코인·채권

#### 면제재산 공제 (2026 기준)
- 퇴직금 일부 (절반 또는 100%)
- 임차보증금 일부 (지역별 상이)
- 예적금 일부 (250만원)
- 보험해약환급금 일부 (250만원)

#### 위험 요소 (청산가치 상승)
- 배우자 재산 50% 합산 (관할법원·회생위원 정책에 따라)
- 최근 1~2년 도박·명품 사치성 소비
- 퇴직금 중간정산 이력
- 가족 명의 분산 재산

### 2.5 채무 처리

| 채무 유형 | 처리 |
|---|---|
| 무담보 (카드·신용대출·캐피탈) | 회생채권. 탕감 대상 |
| 담보 (주담대·자동차담보) | 담보가치 한도 우선변제. 잔여만 회생채권 |
| 세금·건강보험 체납 | 일반 회생채권 제외 가능성 |
| 사채·대부업 | 회생채권 (입증 시) |
| 보증채무 | 별도 처리, 보증인 영향 |

### 2.6 탕감률 산식

```
탕감률 = (총 채무 - 총 변제금) / 총 채무 × 100
```

- 총 변제금: 위 3원칙 중 최댓값
- 변제기간: 기본 36개월, 청산가치 高 시 60개월까지

### 2.7 추정 결과 표현 룰
- **단정 표현 절대 금지**
- **권장 형식**: "약 ○○만 ~ ○○만원 구간", "추정", "개략"
- 결과는 **상한~하한 범위**로 표시 (변동 요인 反映)

## 3. STEP 1: 탭탭탭 (5개 핵심 변수)

### 3.1 입력 항목

| # | 항목 | 형태 | 비고 |
|---|---|---|---|
| 1 | 거주 권역 | 단일선택 (1/2/3/4권역) | 추가주거비 한도 + 기본생계비 |
| 2 | 가구원 수 | 단일선택 (1~5+) | 기본생계비 직접 결정 |
| 3 | 월 소득 (실수령) | 구간선택 (100만 단위) | 가용소득 핵심 |
| 4 | 채무 총액 (3분할 입력) | 구간 입력 × 3 | 무담보 / 담보 / 세금 분리 |
| 5 | 연체 단계 | 단일선택 | HOT 분류 트리거 |

### 3.2 채무 입력 분리 (중요)

```
무담보 채무 (카드·신용대출·캐피탈·사채): [구간 입력]
   - 0~3천만 / 3천~5천 / 5천~1억 / 1억~3억 / 3억 이상

담보 채무 (주담대·자동차담보·전세보증금담보): [구간 입력 + "없음" 옵션]
   - 없음 / 5천만 미만 / 5천~1억 / 1억~3억 / 3억 이상

세금·건강보험 체납: [구간 입력 + "없음" 옵션]
   - 없음 / 500만 미만 / 500~3천 / 3천 이상
```

### 3.3 연체 단계 옵션
- 정상 (연체 없음)
- 연체만 (3개월 이내)
- 장기연체 (3개월 이상)
- 압류·소송 진행 중
- 법원 서류 받음 (지급명령·이행권고 등)

### 3.4 STEP 1 추정 로직 (단순화)

```
입력값 →
1. 기본생계비 = 가구원수별 중위소득 60% 적용
2. 가용소득 = 월소득 - 기본생계비
3. 가용소득 기반 36개월 변제금 = 가용소득 × 36
4. 무담보 채무 - 가용소득 36개월 변제금 = 추정 탕감액
5. 탕감률 = 추정 탕감액 / 무담보 채무 × 100

출력:
- 월 변제액: 약 (가용소득 - 20%) ~ (가용소득 + 0%) 범위
- 탕감률: 약 (탕감률 - 10%) ~ (탕감률 + 10%) 범위

※ 청산가치는 STEP 2에서 정밀 계산
```

### 3.5 STEP 1 결과 화면

```
"○○님의 1차 추정 결과예요"

▶ 추정 월 변제액: 약 ○○만 ~ ○○만원
▶ 추정 탕감률: 약 ○○ ~ ○○%
▶ 변제기간: 36개월

⚠ 이건 핵심 정보 5개로만 계산한 개략 추정이에요.
   추가생계비·재산·부양가족 인정으로 변제금이 더 낮아질 수 있어요.

[자세히 진단받기 (추천)]   [그냥 결과 보기 → 상담 예약]
```

- "자세히 진단받기" → STEP 2 (AI 챗봇)
- "그냥 결과 보기" → STEP 4 (상담 예약 직행, 저가형 타겟 흐름)

## 4. STEP 2: AI 챗봇 (체크박스 + 동적 분기)

### 4.1 챗봇 진입 첫 메시지

```
"방금 입력하신 정보로 1차 추정을 했어요.
 이제 변제금을 더 낮출 수 있는지, 어떤 케이스인지 정밀 확인할게요.
 5분 정도 걸려요.

 ※ 본 진단은 AI가 입력 정보 기반 추정한 개략 금액이며,
   법률 자문이 아닙니다. 정확한 전략은 변호사 검토가 필요합니다."

[시작하기]
```

### 4.2 체크박스 묶음 질문 (한 화면에 일괄)

```
"다음 중 해당되는 것을 모두 골라주세요.
 변제금을 더 낮추거나, 추가 검토가 필요한 부분을 찾기 위함이에요."

━━━ A. 추가생계비 인정 가능성 ━━━
☐ A1. 월세 또는 주담대 갚는 중
☐ A2. 미성년 자녀 있음
☐ A3. 부양 중인 가족 있음 (배우자·부모·성년자녀 등)
☐ A4. 정기 의료비 지출 (본인 또는 가족)
☐ A5. 자녀 교육비·학원비 부담 큼

━━━ B. 직업·소득 구조 ━━━
☐ B1. 전문직 (의사·변호사·세무사·회계사·약사·법무사 등)
☐ B2. 법인 대표 또는 임원
☐ B3. 자영업·프리랜서 (소득 변동 큼)
☐ B4. 4대보험 미가입 (현금 일용직 등)

━━━ C. 재산 ━━━
☐ C1. 부동산 보유
☐ C2. 자동차 보유
☐ C3. 예적금·보험·퇴직금 1천만 이상
☐ C4. 가족 명의 재산 있음

━━━ D. 채무 구조 ━━━
☐ D1. 사채·대부업 있음
☐ D2. 보증인·연대보증 있음
☐ D3. 본인이 다른 사람의 보증인이 되어 있음

━━━ E. 이력·기타 ━━━
☐ E1. 과거 회생·파산 이력 있음
☐ E2. 최근 1년 큰 대출·카드 사용
☐ E3. 최근 1~2년 도박·코인·주식 큰 손실
☐ E4. 퇴직금 중간정산 받음
☐ E5. 명의신탁·차명 거래 있음

☐ 해당 없음 (단순 케이스)
```

| E1 과거 이력 | 언제 / 회생인지 파산인지 / 면책 여부 | 재신청 검토 |
| E2 최근 대출 | 시점 / 사용처 | 기타생계비 불인정 위험 |
| E3 사치성 손실 | 시기 / 규모 | 청산가치 가산 위험 |
| E4 퇴직금 중간정산 | 시점 / 금액 | 청산가치 위험 |
| E5 명의신탁 | 어떤 재산 / 명의자 | 청산가치 + 사기회생 위험 |

### 4.4 챗봇 대화 톤 원칙

| 원칙 | 설명 |
|---|---|
| 단정 금지 | "약 ○○ 구간", "추정", "개략" / "회생 가능합니다" ❌ → "회생 신청 자격 검토 필요" ⭕ |
| 공감 우선 | 채무 액수·연체 단계에 비판·놀람 표현 절대 금지 |
| 빠른 진행 | 자유 대화 길게 끌지 말고 체크박스 + 짧은 후속 질문으로 분류 빠르게 |
| 웰컴 강점 노출 | 결과 화면에서만. 대화 중에는 노출 X (객관성 유지) |
| 상담 유도는 자연스럽게 | "이 부분은 양홍수 변호사님이 정확히 봐드릴 수 있어요" |
| 법률 판단 금지 | "○○하면 됩니다" ❌ / "이런 경우 ○○ 검토가 필요해요" ⭕ |

### 4.5 자동 Pro 트리거 (대화 중 즉시 분류)

다음 중 하나라도 감지 시 케이스 분류 = **Pro**

- B1 전문직 체크
- B2 법인 대표 체크
- B3 자영업 + 매출 큰 변동
- 무담보 채무 1억 이상
- C1 부동산 보유 + 시가 5천 이상
- C4 가족 명의 재산 (명의신탁 의심)
- D2 보증인 있음
- E1 과거 회생/파산 이력
- E3 도박·코인 사치성 손실
- E4 퇴직금 중간정산
- E5 명의신탁

### 4.6 자동 HOT 트리거 (즉시 변호사 핫라인)

다음 중 하나라도 감지 시 = **HOT (30분 내 변호사 직접 전화 옵션 추가)**

- 연체 단계: "법원 서류 받음" 또는 "압류·소송"
- 사용자 입력 키워드: "압류 진행 중", "통장 압류", "급여 압류", "경매", "강제집행 통지"

## 7. CRM 진입 분기

### 7.1 진단기 이탈자 (전화번호 미수집)

- 별도 CRM 진입 없음

- CRM 진입: `lead_pdf_only`
- 시퀀스: D+0 PDF → D+2 회생 가이드 → D+5 사례 → D+9 압류 단계별 가이드 → D+14 상담 10% 할인 (또는 노쇼방지비 면제) → D+30 마지막 안내

### 7.3 결제 후 노쇼
- CRM 태그: `no_show`
- 시퀀스: 당일 사과 메시지 + 재예약 링크 (노쇼방지비 환불 불가 안내)

### 7.4 상담 완료 후 수임 보류
- CRM 태그: `consulted_pending`
- 시퀀스: D+3 재확인 → D+7 결정 가이드 → D+14 추가 상담 제안 → 월 1회 유지
- 6~12개월 장기 유지 (트리거 발생 시 즉시 전환)

| 결제 (상담 예약) 전환율 | 5~15% | PG |
| 노쇼율 | 5~10% 이하 | 운영 |
| 상담 → 수임률 | 40~60% | 운영 |
| CPSale (수임 1건당 광고비) | 케이스별 목표 설정 | 메타 + 운영 |
| LTV (수임료 평균) | 케이스별 측정 | 운영 |

## 부록 A: 변제금 계산 의사코드 (백엔드)

```python
def estimate_variation(
    region: int,           # 1~4권역
    household_size: int,   # 1~5+
    monthly_income: int,   # 월 실수령
    debt_unsecured: int,   # 무담보 채무
    debt_secured: int,     # 담보 채무
    debt_tax: int,         # 세금 체납
    delinquency_status: str,  # 정상/연체/장기연체/소송/법원서류
    additional_factors: dict = None  # STEP 2 결과
) -> dict:

    # 1. 기본생계비 (2026 기준)
    base_living_cost = {
        1: 1538543, 2: 2519575, 3: 3215422,
        4: 3896843, 5: 4534031, 6: 5133571
    }[household_size]

    # 2. 추가생계비 (STEP 2 반영)
    additional = 0
    if additional_factors:
        additional += calc_housing_extra(region, additional_factors)
        additional += calc_education_extra(additional_factors)
        additional += calc_medical_extra(household_size, additional_factors)
        additional += calc_dependent_extra(additional_factors)

    # 3. 가용소득
    available_income = monthly_income - base_living_cost - additional
    available_income = max(available_income, 0)

    # 4. 36개월 변제금 (가용소득 기준)
    repayment_36m = available_income * 36

    # 5. 청산가치 (STEP 2 반영, STEP 1만이면 0으로 간주)
    liquidation_value = 0
    if additional_factors:
        liquidation_value = calc_liquidation(additional_factors)

    # 6. 총 변제금 (3원칙 중 최댓값)
    total_repayment = max(
        repayment_36m,
        liquidation_value,
        # 최저변제액은 케이스별 다름
    )

    # 7. 변제기간 결정
    if total_repayment > repayment_36m:
        # 청산가치가 더 큼 → 60개월로 연장 검토
        monthly_repayment = total_repayment / 60
        period = 60
    else:
        monthly_repayment = available_income
        period = 36

    # 8. 탕감률
    discharge_rate = (debt_unsecured - total_repayment) / debt_unsecured * 100
    discharge_rate = max(min(discharge_rate, 95), 0)

    # 9. 결과 (범위로 출력)
    return {
        "monthly_repayment_min": int(monthly_repayment * 0.8),
        "monthly_repayment_max": int(monthly_repayment * 1.0),
        "discharge_rate_min": discharge_rate - 10,
        "discharge_rate_max": discharge_rate + 10,
        "period_months": period,
        "liquidation_value": liquidation_value,
        "case_classification": classify_case(additional_factors, debt_unsecured),
        "is_hot": is_hot_case(delinquency_status, additional_factors),
    }


def classify_case(factors: dict, debt_unsecured: int) -> str:
    """Lite / Pro 분류"""
    if not factors:
        return "lite"

    pro_triggers = [
        factors.get("B1_professional"),
        factors.get("B2_corporate"),
        factors.get("B3_self_employed_volatile"),
        debt_unsecured >= 100_000_000,
        factors.get("C1_real_estate"),
        factors.get("C4_family_assets"),
        factors.get("D2_guarantor"),
        factors.get("E1_history"),
        factors.get("E3_gambling_loss"),
        factors.get("E4_severance_intermediate"),
        factors.get("E5_nominee"),
    ]

    if any(pro_triggers) or sum_checked(factors) >= 3:
        return "pro"
    return "lite"


def is_hot_case(status: str, factors: dict) -> bool:
    """HOT 핫라인 트리거"""
    return (
        status in ["압류·소송", "법원서류"]
        or factors.get("hot_keywords_matched", False)
    )
```
